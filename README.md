# SyncPro (교육청 IoT 충전함 통합 관리 서비스)

> 본 프로젝트는 공공/보안 환경에서 운영되어  
> **회사 정책상 전체 소스코드 반출이 불가능합니다.**  
> 제가 담당한 **역할·기술적 의사결정·구현 구조**를 중심으로 설명합니다.

---

## 개요
교육청 → 교육지원청 → 학교 단위로 보급되는 **IoT 태블릿/휴대기기 충전함**을  
실시간으로 모니터링/제어할 수 있는 **관제 시스템 + 교사용 모바일 앱**입니다.

본 서비스는 공공기관 환경에서 **다수의 IoT 디바이스를 단일 플랫폼으로 관리**하는 데 목적이 있으며,  
운영 효율, 장애 대응 속도, 유지보수 편의를 크게 개선했습니다.

---

## 디바이스 등록 & 네트워크 구조

충전함은 각 학교 내부 **Wi-Fi(AP)** 에 연결되어야 관제 서버와 통신할 수 있습니다.

1. **학교 AP 선택**  
2. **충전함 내부 Wi-Fi 모듈에 AP 정보 전달**
3. **시리얼(고유번호) 기반 장비-학교-교육지원청 계층 매핑**
4. **MQTT + REST API 통신으로 상태/제어 연결 활성화**

> 이 구조를 통해 **각 충전함은 학교 단위로 독립 운영되며**,  
> 상위 기관(교육청)은 전체 단위 모니터링을 수행할 수 있습니다.

---

## 시스템 구성

### 1) 관리자 관제 웹 (ASP.NET MVC)
| 기능 | 설명 |
|---|---|
| 생산/출고 관리 | 시리얼 발급 → 배송/설치 이력 관리 |
| 기관별 장비 현황 | 교육청/교육지원청/학교 단위 트리 구조 관리 |
| 실시간 상태 모니터링 | 온도,네트워크,충전 포트 상태 조회 |
| 원격 제어 | 윤번(순차 충전), 전원 제어, 장비 재부팅 |
| 장애/AS 처리 | 접수 → 처리 이력 기록 |

### 2) 교사용 모바일앱 (React Native)
| 기능 | 설명 |
|---|---|
| 실시간 충전현황 조회 | 포트별 충전 상태 표시 |
| 원격 제어 | 현장에서 전원/충전 모드 조작 가능 |
| 이상 알림 | 과열 / 통신 오류 감지 시 알림 |

---

## 데이터 처리 구조
- IoT 장비 데이터 → **MQTT 소켓 프로토콜**
- 실시간 RAW 데이터 저장 → **MongoDB**
- 관제 & 앱 UI 제공 → **가공/요약 데이터 변환 Layer 구현**

---

## 담당한 역할 (내 기여도)

| 역할 | 내용 |
|---|---|
| 웹 관제 개발 | 관리자 웹 전 화면 / 기능 개발 (ASP.NET MVC) |
| 실시간 데이터 처리 | MQTT 수신 → MongoDB 저장 → UI 데이터 가공 로직 설계 |
| 디바이스 제어 로직 | 윤번충전/포트 전원/장비 재부팅 등 소켓 명령 처리 |
| RN 앱 상태 화면 개발 | 충전함 상태 UI/UX 및 네트워크 매핑 처리 |
| 운영 자동화 | 장애 감지 로직 / 상태 알림 흐름 개선 |
